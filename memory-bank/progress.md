## 当前最大风险
- 原型的视觉完成度高，容易误判为“已接近完成”，但核心数据层尚未建立。
- 如果不先做落库与职责拆分，后续继续堆功能会让插件难以维护。
- 如果不把第三方接口迁到服务端，正式环境会存在密钥泄露风险。
- 如果后续继续同时维护 `@...` 和无 `@` 两套正文，memory-bank 会再次分叉；必须坚持“正文只维护一份”。
- 本机 CLI PHP 是新版本，但 Web 运行时更老；如果以后只用 CLI 验证语法，仍可能漏掉线上 parse error。
- `showDock`、`bgBlur`、`simpleMode`、默认搜索引擎、壁纸选择仍只停留在前端状态；登录入口补上后，应尽快进入设置持久化。
- 删除用户时，插件当前不会自动级联清理该用户的个人桌面；这不会影响当前 MVP 主链路，但后续如果增加后台用户清理或数据回收功能，需要显式补这一层。
